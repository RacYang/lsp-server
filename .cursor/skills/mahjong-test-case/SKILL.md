# 麻将测试用例

## When to use

当需要新增或更新 YAML 驱动的麻将示例，用于复现摸牌顺序、和牌检测、番种计分、退税/包牌或房间动作解析时使用。

## Inputs

- 目标行为：算法、番种、结算或房间动作。
- 牌张输入：清晰的万筒条字符串与座位上下文。
- 期望输出：和牌结果、番种分解、分数流水或动作响应。

## Steps

1. 在 YAML 中用清晰的牌张字符串描述夹具，避免依赖随机牌墙。
2. 编码触发进张、出牌、抢答或结算动作，保持执行顺序可回放。
3. 同时断言结果与适用时的番种分解、罚分、退税或包牌标记。
4. 为边界条件补反例，避免只验证快乐路径。
5. 如新增字段，更新读取 fixture 的测试代码与相关文档。

## Verify

- 运行相关包测试，例如 `go test ./internal/mahjong/...` 或 `go test ./internal/service/room/...`。
- 最后运行 `make verify-fast`。
