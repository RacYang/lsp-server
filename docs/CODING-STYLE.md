# 编码风格

## Go

- 倾向小而职责清晰的包，依赖关系显式可追踪。
- 返回错误时使用 `%w` 包装，保留错误链。
- 对构成模块边界的导出 API 保持文档注释（见 [ADR-0005](adr/0005-comment-system.md)）。
- 传输层与基础设施代码不得侵入 `domain` 与 `mahjong` 等纯领域包。

## 注释体系（规范）

与 [ADR-0005](adr/0005-comment-system.md) 对齐：

- **包注释**：说明职责、边界与误用后果。
- **类型 / 函数注释**：导出符号必须可读地说明契约；复杂非导出符号在行为不显然时补充。
- **关键分支**：并发、状态机、协议兼容、性能与规则分支优先写「为什么」。
- **语言**：注释正文以简体中文为主；技术关键词可保留英文（见 SSOT `commenting.technical_keywords`）。

## 文档

- 文档解释**决策与约束**，而非琐碎实现步骤堆砌。
- ADR 记录不可逆或与治理强相关的选择。

## 生成代码

生成文件必须带标准头注释，且禁止手工编辑；生成路径不参与注释语言占比（见 SSOT `commenting.code_exclude`）。

## 错误字符串

`errors.New` / `fmt.Errorf` 面向调用方与监控的字符串使用**英文小写**、无尾标点，与 [ADR-0004](adr/0004-language-and-writing-policy.md) 一致。
