# 协议

## 帧布局

每个客户端帧使用**定长二进制头**后接 **Protobuf** 载荷。

```text
+----+------+--------+-----------------+
| 4B | 2B   | 4B     | N bytes         |
| len| msgID| seqID  | protobuf payload|
+----+------+--------+-----------------+
```

## 原则

1. 消息 ID 稳定并在 proto 注释中说明。
2. RPC proto 与客户端 proto 分树维护。
3. 仅在 `proto-baseline` 存在后启用破坏性检查。

## 传输保证

- 客户端传输在单连接上有序。
- 当消息携带序列 ID 与令牌时，请求具备幂等语义。
- 服务端推送使用相同的帧封装。
