# 规则引擎

## 目标

麻将变体规则必须可替换，而无需重写传输、存储或房间编排。

## 设计

规则接口负责：

- 牌墙构建
- 开局动作（如定缺、换三张）
- 动作许可（如碰、杠、和）
- 计分与结算行为

## 注册表

变体按名称注册并通过配置选择。房间通过**接口**请求规则行为，而非按规则名分支。

## 首个变体

`sichuan_xzdd` 为首个实现，支持：

- 定缺
- 无吃牌
- 和牌后血战续行
- 自摸、点炮胡与抢杠胡上下文
- 基础番种、海底/杠上/根与结构化结算

房间编排负责 actor 串行化、等待态、托管超时与持久化恢复；规则包不得依赖传输层、存储层或 room service。
