# ADR-0034 日志动态级别管控

## 状态

已采纳。

## 背景

生产排障常需要短时间提高日志级别，但开放无鉴权 HTTP 写端点会带来风险：任何能访问观测端口的主体都可能把日志调到 debug，造成噪声、成本和敏感信息暴露。

## 决策

Phase 1 仅在 `pkg/logx` 提供 `AtomicLevel` Go API，供进程内组件安全切换日志级别。本轮不挂载 `/debug/log/level` HTTP 端点。

若未来需要 HTTP 或信号接线，必须先明确鉴权、审计和访问边界，并与 ADR-0027 的凭据管理策略一致。

## 后果

本轮获得可测试的动态级别基础能力，同时避免把未鉴权写接口暴露到观测面。
