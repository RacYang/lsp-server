# ADR-0005 注释体系

## 决策

在 ADR-0004 要求「注释以中文为主」的前提下，本 ADR 约定**注释分层**与**必须注释**的触发条件，优先解释「为什么」而非复述代码行为。

## 注释分层

1. **包注释**：说明包职责、边界、禁止事项；导出较少的实现包可简短，但不得为空泛套话。
2. **类型注释**：导出类型若承载不变式、状态或跨包契约，必须说明语义与误用后果。
3. **函数注释**：导出函数说明参数、返回值、错误语义；非导出函数在行为非显而易见时补充。
4. **关键分支注释**：涉及并发、重试、协议兼容、性能热路径、与规则引擎相关的分支，应点明设计取舍。

## 必须写注释的典型场景

- 对外 API 或跨模块边界（handler、service、领域服务）。
- 状态机迁移、房间生命周期、断线重连与幂等等语义。
- 非显而易见的性能优化、锁顺序、避免死锁的约束。
- 与安全、资金、作弊风险相关的校验分支（未来扩展时适用）。

## 与硬检查的关系

- 中文占比与排除路径由 SSOT `commenting` 节驱动，enforcer 为 `scripts/verify-lang-comments.py`。
- **不**在 Phase 0 引入「注释覆盖率」类指标，以免噪音；以规范与评审为主。

## 后果

团队对注释位置与深度有统一预期；硬检查聚焦语言与明显遗漏的高风险区，后续可迭代 AST 级辅助工具。
