# ADR-0026 PostgreSQL 备份与恢复策略

## 状态

已采纳。预发环境恢复演练已完成，执行编号为 `restore-phase6-preprod-20260428`，详见 `bench/restore-phase6-preprod-20260428/summary.md`。

## 背景

ADR-0013 引入 PostgreSQL 事件日志与结算持久化，ADR-0024 将部署形态推进到生产交付。
但当前只规定了"PostgreSQL 单副本起步 + 定期备份"，没有明确 RPO、RTO 与恢复演练流程。

## 决策

### 1. 备份形态

- 生产环境启用每日全量备份与 WAL 归档。
- 预发环境至少保留每日全量备份，用于验证恢复脚本与 schema 演进。
- 备份文件进入对象存储或托管数据库备份系统，不进入 Git 仓库。

### 2. 恢复目标

| 维度 | 目标 |
| --- | --- |
| RPO | ≤ 15 分钟 |
| RTO | ≤ 60 分钟 |
| 恢复演练 | 每月至少一次 |
| 保留周期 | 生产 30 天，预发 7 天 |

### 3. 演练剧本

1. 从最新全量备份恢复到临时 PostgreSQL 实例。
2. 回放 WAL 到指定时间点。
3. 启动 `room` 进程指向临时实例。
4. 使用 `SnapshotRoom` 与 `StreamEvents` 校验房间事件可读。
5. 输出恢复报告，记录恢复耗时、数据缺口与异常。

## 后果

- 恢复 runbook 已落地到 `deploy/ops/postgres-restore.md`，后续演练沿用同一报告模板。
- 恢复演练不进入默认 `make verify`；CI 只校验脚本文法和文档格式。
- 若托管数据库提供 PITR，优先复用托管能力，不在应用层重复造备份系统。
