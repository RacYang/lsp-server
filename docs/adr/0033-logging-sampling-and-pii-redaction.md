# ADR-0033 日志采样与 PII 脱敏策略

## 状态

已采纳。

## 背景

房间、重连、限流与存储路径未来会产生大量可观测数据。高频运行态应进入指标，日志只承载关键业务里程碑、异常、降级与审计信息。与此同时，日志必须避免把 token、密码、邮箱、手机号、IP 等敏感信息写入采集链路。

## 决策

`pkg/logx` 提供采样框架，但默认关闭。即使未来开启采样，`Error` 级日志也不得被采样丢弃。

PII 治理 Phase 1 仅做键名层约束：`logging.pii_redact_keys` 中的键不得出现在业务日志调用中；门面输出阶段会按键名将命中字段替换为 `***`。值层正则、嵌套结构遍历与更强识别策略不在本 ADR 范围内。

## 后果

采样能力可在容量与 SLO 讨论成熟后启用，不会提前牺牲错误可见性。PII 键名治理能先挡住最常见误用，但不能被视为完整隐私合规方案。
