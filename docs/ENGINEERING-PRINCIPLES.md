# 工程原则

## 范围

本文描述仓库级工程不变量。

## 原则

1. 验证类命令**不得**修改仓库文件。
2. `.build/config.yaml` 是唯一由人编写的治理源（SSOT）。
3. 派生文件纳入版本控制，并由基于内容的校验保护。
4. 约束类规则必须可执行，并由负例样本背书。
5. 工具版本由 SSOT 约束。
6. 阶段晋升须显式且可评审。

## 运行模式

- `bootstrap`、`generate`、`fix`、`fix-file` 允许改动工作区。
- 所有 `verify-*` 目标必须保持只读。

## 评审标准

任何引入**新硬约束**的提案须同时包含：

1. 一份 ADR。
2. `.build/config.yaml` 中的对应条目。
3. 生成或运行时的 enforcer。
4. 证明 enforcement 生效的负例样本。
