# 麻将算法

## 包划分

- `tile`：牌张标识与解析辅助。
- `wall`：确定性牌墙构建与摸牌。
- `hand`：手牌与副露。
- `hu`：和牌型校验。
- `fan`：番种与计分分量。

## 四川血战当前覆盖

- 和牌判定支持标准和形与七对；规则层通过 `HuContext` 区分自摸、点炮与抢杠来源。
- 番种分解覆盖平胡、对对胡、清一色、七对、根、杠上开花、海底、海底炮与抢杠胡等主链路标签。
- 血战轮转由房间引擎记录已胡座位，继续推进到三家胡或牌墙耗尽。
- 杠流水通过 `GangRecord` 进入结算上下文，用于刮风下雨、抢杠与退税等后续细化。

## 测试策略

算法通过 **YAML** 夹具验证。每个用例声明：

- 初始手牌
- 进张或动作输入
- 期望的和牌或动作结果
- 适用时的期望番种分解

## 确定性

牌墙创建必须由**种子**驱动，以便从日志回放整局。
